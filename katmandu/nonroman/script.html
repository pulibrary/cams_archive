Misc. inputting points:

Workflow for original and member copy:

  1. Complete the entire record in OCLC.  Keep it in your local save file while you are working on it.  Make sure you are completely done with the roman and the non-roman fields before moving to the next step.
  2. Update holdings
  3. Open the Princeton IR
  4. Export
  5. Go to Voyager and search for the brief record
  6. open the OCLC record from the import file and Overlay the brief record
  7. Add 902  (do NOT mark the record for export)
  8. Add mfhd and item records

For added volumes:

records without script:

  • if the bib needs to be changed: work in Voyager and mark the record for export
  • if only the mfhd is changed: work in Voyager but do not export the record
records with script:
  • if the bib needs to be changed: work in OCLC, bring in and overlay the updated bib, add holdings
  • if only the mfhd needs to be changed: work in Voyager

Decisions about how to enter data

personal name headings if the tool converts the date correctly, leave it adding the hyphen if necessary; if the tool does not convert the date properly, remove the date entirely
260 dates use the form the tool provides for simple dates; for complex statements, delete what the tool provides and copy |c from the romanized field
245 et al.

no decision yet

260 S.l. and s.n. no decision yet
111/711 qualifiers delete what the tool provides and copy the |n |d and |c from the romanized field
110/610/710 qualifiers

(city, country) qualifiers should be removed if the tool does not convert them properly

260 subfield a always provide an original script version of the place of publication even when the romanized field has [Cairo], [Tehran], etc.
