MARC validation errors and how to deal with them

When you save a record to the database a MARC Error Report like the screen below will appear if there are any MARC errors present and if your MARC validation is active.

The 3 types of errors that will come up most frequently are in this example:


INVALID VALUE

                Look at the indicator in question and fix it.  In some cases it may be an indication that the entire field needs attention--an invalid value of 4 in a 650 field will help to remind us to delete non-LCSH subject headings.

MANDATORY ... NOT FOUND

                Add the field in question.  If you aren't sure what belongs in the field use the F2 key to find a description of the field, indicators and subfields.

... NOT DEFINED IN YOUR TAG TABLE

                Delete the field in question.  If a field isn't defined it means we don't want it.

If you think that the MARC error report is wrong or won't allow you to do something that you need to do, consult your supervisor.
